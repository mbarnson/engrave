<objective>
Implement the LilyPond compile-check-fix retry loop: subprocess compilation, structured error parsing, LLM-assisted error recovery with deduplication, and wire the pipeline through the CLI.

Purpose: FNDN-05 requires automatic error recovery for LilyPond compilation. This is the core feedback loop that makes LLM-generated LilyPond viable -- without it, first-attempt compilation failures would require manual intervention. The error parser and fix loop have clearly defined I/O, making them TDD candidates.

Output: Working compile-fix pipeline invokable via `engrave compile <file.ly>`, with unit tests for each component and Gherkin integration tests for end-to-end behavior.
</objective>
