<objective>
Build the ingestion pipeline that takes raw score files from Mutopia and PDMX, validates/compiles them, chunks them into phrases, and indexes them into ChromaDB.

Purpose: This is the ETL layer that populates the corpus. Mutopia provides ~2,124 LilyPond scores directly; PDMX provides ~254K MusicXML scores that are converted via musicxml2ly. Together they produce the 10K+ retrievable phrase examples that Phase 3 needs for few-shot code generation.

Output: Working ingestion pipeline with Mutopia and PDMX adapters, tested with fixture files.
</objective>

    6. Create `src/engrave/corpus/ingest/pipeline.py`:
       - `@dataclass IngestionResult`: source_path (Path), chunks_indexed (int), midi_generated (bool), errors (list[str]), skipped (bool), skip_reason (str|None)
       - `async def ingest_score(ly_source: str, source_path: Path, source_collection: str, header_metadata: dict, compiler: LilyPondCompiler, store: CorpusStore, router: InferenceRouter | None = None) -> IngestionResult`:
         a. Quality filter: skip if source is <10 characters or contains fewer than 2 notes (degenerate case per user decision)
         b. Call ensure_midi_block to inject `\midi {}` if missing
         c. Compile with LilyPondCompiler. If fails and router is provided, run through compile_with_fix_loop (per user decision). If still fails, skip and log with reason.
         d. Check for MIDI output file (.midi). Set midi_generated flag.
         e. Call chunk_score from Plan 02's chunker with the compiled source, source_path, source_collection, header_metadata
         f. For each raw chunk dict, call extract_metadata and generate_description from Plan 02
         g. Build Chunk model objects with full ScoreMetadata
         h. If MIDI was generated and mido is available, extract MIDI features (velocity histogram, pitch range, rhythmic complexity) and attach to chunk.midi_features
         i. Call store.add_chunks() to index all chunks
         j. Return IngestionResult with counts

       - `async def ingest_mutopia_corpus(repo_path: Path, compiler: LilyPondCompiler, store: CorpusStore, router: InferenceRouter | None = None, max_scores: int | None = None) -> list[IngestionResult]`:
         a. Call discover_mutopia_scores to find all .ly files
         b. For each score (up to max_scores if set): read source, extract Mutopia header, call ingest_score with source_collection="mutopia"
         c. Log progress every 100 scores
         d. Return list of IngestionResult

    7. Create `tests/fixtures/corpus/mutopia_bach.ly` — a realistic Mutopia-format LilyPond file with proper `\header` block containing Mutopia fields, `\layout {}`, no `\midi {}` (to test injection), ~16 bars with a rehearsal mark.
