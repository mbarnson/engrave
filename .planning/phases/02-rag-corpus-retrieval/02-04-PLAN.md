<objective>
Build the retrieval interface that Phase 3 will use to get few-shot LilyPond examples for code generation.

Purpose: This is the consumer-facing layer of the corpus. Phase 3's generation pipeline will call `from engrave.corpus import retrieve` to get relevant examples. The hybrid scoring (metadata filter + embedding similarity) ensures results are both structurally appropriate (right instrument, style) and stylistically similar (embedding match).

Output: Working retrieval API accessible as both Python library and CLI command, with integration tests using a pre-populated test corpus.
</objective>
