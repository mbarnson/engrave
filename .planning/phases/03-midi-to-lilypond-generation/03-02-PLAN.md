    **TDD: RED phase first, then GREEN.**

    Create `src/engrave/generation/__init__.py` re-exporting public API.

    **coherence.py**: Pydantic model for musical coherence state passed between section-generation LLM calls:
    - `CoherenceState(BaseModel)` with fields per user decision:
      - section_index: int = 0
      - total_sections: int = 1
      - key_signature: str = "c \\major" (LilyPond format)
      - time_signature: str = "4/4"
      - tempo_bpm: int = 120
      - dynamic_levels: dict[str, str] = {} (track_name -> pp/p/mp/mf/f/ff)
      - articulation_style: str = "" (e.g. "legato", "marcato-heavy")
      - rhythmic_density: str = "moderate" ("sparse"/"moderate"/"dense")
      - voicing_patterns: list[str] = [] (e.g. "close voicing", "drop-2")
      - open_ties: dict[str, list[str]] = {} (track_name -> tied pitch names from prev section)
      - last_bar_summary: str = "" (brief text of final bar content)
      - generated_summary: str = "" (running summary, capped at ~300 tokens / ~1200 chars)
    - `to_prompt_text() -> str`: Serialize to compact, human-readable text for LLM prompt inclusion. Only include non-empty/non-default fields.
    - `update_from_section(section_ly: str, section_midi_text: str) -> CoherenceState`: Create updated state from generated LilyPond output. Increment section_index, update dynamic_levels from last dynamic marking in LilyPond, detect open ties (notes with ~ at section end), append to generated_summary (with compression if over char limit -- simple truncation of oldest content for now, LLM compression deferred per open question #3).
    - `initial_from_analysis(analysis) -> CoherenceState`: Class method to create initial state from MidiAnalysis. Takes key, time sig, tempo from analysis.

    **templates.py**: LilyPond structural template generation:
    - `build_score_template(instrument_names: list[str], section_label: str, start_bar: int, end_bar: int) -> str`: Generate complete LilyPond score skeleton with:
      - \version "2.24.4" header
      - Comment with section label and bar range
      - One variable declaration per instrument (e.g., `instrumentOne = { }`)
      - \score block with \new Staff for each instrument referencing its variable
      - ABSOLUTE pitch mode -- NO \relative anywhere
      - Concert pitch -- NO \transpose anywhere
      - Variable names: sanitize instrument names to valid LilyPond identifiers (camelCase, no spaces/special chars)
    - `build_instrument_variable(var_name: str, music_content: str) -> str`: Format a single instrument variable with its music content filled in.
    - `sanitize_var_name(instrument_name: str) -> str`: Convert instrument name to valid LilyPond variable name (e.g., "Alto Sax 1" -> "altoSaxOne", "Trumpet in Bb" -> "trumpetInBb").
    - `extract_variable_names(template: str) -> list[str]`: Parse template to find variable names the LLM must fill.

    **Unit tests** (test_coherence.py):
    - test_default_coherence_state: default values are sensible
    - test_to_prompt_text_includes_key_tempo_time: basic fields present in output

    - test_to_prompt_text_includes_open_ties: open ties appear when set

    - test_update_from_section_increments_index: section_index advances
    - test_update_from_section_detects_ties: notes ending with ~ detected

    - test_initial_from_analysis: creates state from analysis data

    **Unit tests** (test_templates.py):

    - test_build_score_template_section_comment: section label and bar range in comment
    - test_sanitize_var_name: various instrument names produce valid LilyPond identifiers
    - test_extract_variable_names: correctly parses variable names from template

    **TDD: RED phase first, then GREEN.**

    **prompts.py**: Prompt construction with explicit budget management:
    - `PromptBudget` dataclass with token allocations:
      - system_instructions: int = 2000 (fixed)
      - template_tokens: int = 500 (fixed)
      - coherence_tokens: int = 500
      - rag_tokens: int = 3000
      - midi_tokens: int = 4000
      - output_reserve: int = 8000
      - safety_margin: int = 4000
      - total_limit: int = 32000
    - `estimate_tokens(text: str) -> int`: Simple token estimation (chars / 4 as rough approximation, sufficient for budget management).
    - `fit_within_budget(budget: PromptBudget, midi_text: str, rag_examples: list[str], coherence_text: str) -> tuple[str, list[str], str]`: Truncate components to fit budget. Priority: (1) reduce RAG examples from N to fewer, (2) truncate coherence to essentials (key, time sig, tempo, dynamics only), (3) truncate MIDI text (remove least-populated instruments). Return the fitted (midi_text, rag_examples, coherence_text).
    - `build_section_prompt(section_midi: dict[str, str], coherence: CoherenceState, rag_examples: list[str], template: str, budget: PromptBudget | None = None) -> str`: Assemble the complete prompt. Apply budget fitting. Format sections: RULES, CURRENT MUSICAL STATE, LILYPOND TEMPLATE, SIMILAR EXAMPLES, MIDI CONTENT, and instruction to generate music content for each variable.
    - The RULES section must explicitly state: (1) absolute pitch mode, (2) music content only -- no \version/\score/\new Staff, (3) concert pitch, (4) preserve all MIDI musical content, (5) add appropriate articulations/dynamics.

    **failure_log.py**: Structured failure logging for TUNE-02 fine-tuning:
    - `FailureRecord(BaseModel)` with fields:
      - timestamp: str (ISO 8601)
      - section_index: int
      - midi_token_text: str (the MIDI tokens that were in the prompt)
      - prompt_sent: str (full prompt text)
      - lilypond_error: str (compiler error output)
      - lilypond_source: str (the failing LilyPond code)
      - retry_attempts: int
      - error_hashes: list[str] (SHA256 hashes of each attempt's error)
      - coherence_state: dict (serialized CoherenceState at time of failure)
    - `log_failure(record: FailureRecord, log_dir: str = ".engrave/failures") -> Path`: Write failure record as JSON file with timestamp-based filename. Create log_dir if not exists.
    - `load_failure_log(log_dir: str = ".engrave/failures") -> list[FailureRecord]`: Load all failure records from log directory.

    **Unit tests** (test_prompt_budget.py):
    - test_estimate_tokens_rough_accuracy: token estimate within 2x of actual for typical text
    - test_fit_within_budget_no_truncation: components within budget returned unchanged
    - test_fit_within_budget_reduces_rag_first: when over budget, RAG examples reduced before other components
    - test_fit_within_budget_truncates_coherence_second: after RAG reduction, coherence truncated
    - test_fit_within_budget_truncates_midi_last: MIDI text truncated only as last resort
    - test_build_section_prompt_has_all_sections: output contains RULES, MUSICAL STATE, TEMPLATE, EXAMPLES, MIDI sections
    - test_build_section_prompt_rules_absolute_pitch: RULES section mentions absolute pitch
    - test_build_section_prompt_rules_concert_pitch: RULES section mentions concert pitch
    - test_build_section_prompt_rules_no_score_block: RULES section says not to generate \score

    **Unit tests** (test_failure_log.py):
    - test_log_failure_creates_file: JSON file written to log_dir
    - test_log_failure_creates_dir: log_dir created if missing
    - test_log_failure_content_valid: file content deserializes to FailureRecord
    - test_load_failure_log_reads_all: multiple records loaded and returned
    - test_load_failure_log_empty_dir: empty dir returns empty list
    - test_failure_record_serialization: FailureRecord round-trips through JSON
