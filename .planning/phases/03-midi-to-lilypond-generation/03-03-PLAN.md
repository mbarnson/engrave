  <action>
    **pipeline.py**: Orchestrate end-to-end MIDI-to-LilyPond generation:

    - `GenerationResult` dataclass: success (bool), ly_source (str -- complete assembled .ly file or empty), sections_completed (int), total_sections (int), failure_record (FailureRecord | None)
    - `GenerationHaltError(Exception)`: raised when a section fails after all retries. Carries section index, final errors, and attempt count.

    - `async generate_from_midi(midi_path: str, router: InferenceRouter, compiler: LilyPondCompiler, rag_retriever=None, user_labels: dict[int, str] | None = None) -> GenerationResult`:
      1. Load and normalize MIDI: `load_midi(midi_path)` -> tracks + metadata
      2. Analyze musical properties: `analyze_midi(midi_path)` -> key, tempo, time sig, instruments
      3. If user_labels provided, override track instrument_name fields. If not provided and tracks lack metadata, log a warning about ambiguous instrument assignment.
      4. Detect section boundaries: `detect_sections(midi_path)` -> list of SectionBoundary
      5. Initialize CoherenceState from analysis: `coherence = CoherenceState.initial_from_analysis(analysis)`
      6. Build instrument_names list from tracks (using user_labels, track metadata, or generic names)
      7. For each section:
         a. Filter notes per track for this section's bar range
         b. Tokenize MIDI for each track: `tokenize_section_for_prompt(...)`
         c. Query RAG for similar examples: construct query as `f'{analysis.key_signature} {analysis.tempo_bpm}bpm {", ".join(instrument_names)}'` and call `rag_retriever.retrieve(query, limit=3) if rag_retriever else []`
         d. Build LilyPond template for this section: `build_score_template(instrument_names, section_label, start_bar, end_bar)`
         e. Build prompt: `build_section_prompt(section_midi, coherence, rag_examples, template)`
         f. Call LLM via router: `await router.complete(role="generator", messages=[...], temperature=0.3)`
         g. Extract LilyPond from response: `extract_lilypond_from_response(response)` (from Phase 1 fixer module)
         h. Parse LLM response to extract music content per instrument variable using `parse_instrument_blocks(llm_response)` from templates.py. This returns a `dict[str, str]` mapping variable names to music content strings.
         i. Fill template variables with extracted music content using `build_instrument_variable(var_name, content)` for each instrument.
         j. Compile through fix loop: `await compile_with_fix_loop(source, router, compiler)` (from Phase 1)
         k. If compilation fails: `log_failure(...)`, return GenerationResult(success=False, ..., failure_record=record)
         l. Update coherence state (captures returned new state): `coherence = coherence.update_from_section(compiled_source, midi_text)`
         m. Store compiled section source
      8. Assemble all section sources into complete .ly file: `assemble_sections(...)`
      9. Return GenerationResult(success=True, ly_source=assembled, sections_completed=N, total_sections=N)

    - `async generate_section(...)`: Extract the per-section logic into a helper for testability. Takes section MIDI tokens, coherence, RAG examples, template, router, compiler. Returns (section_ly_source, updated_coherence).

    **assembler.py**: Assemble section outputs into a complete .ly file:

    - `assemble_sections(section_sources: list[str], instrument_names: list[str], analysis: MidiAnalysis) -> str`:
      1. Generate a single master \version header and global settings block
      2. For each instrument, concatenate the music content from all sections into one variable
      3. Build a single \score block with one \new Staff per instrument
      4. Include tempo, key, and time signature in the global context
      5. Return the complete, self-contained .ly file string
    - The assembled file must compile as a single unit (not section-by-section). The assembly merges section music into continuous instrument variables.
    - Handle section continuity: if a section ends with a tie (~), the next section's music starts tied.
    - Include metadata comments: generation timestamp, source MIDI filename, number of sections.
  </action>
