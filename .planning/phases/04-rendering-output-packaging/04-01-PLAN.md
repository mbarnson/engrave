Create `src/engrave/rendering/stylesheet.py` with string constants for LilyPond `\paper` and `\layout` blocks:

**Conductor score constants:**
- `CONDUCTOR_SCORE_PAPER`: tabloid landscape `#(set-paper-size "tabloid" 'landscape)`, appropriate margins
- `CONDUCTOR_SCORE_LAYOUT`: `\RemoveEmptyStaves` with first-system exception, `Keep_alive_together_engraver` for StaffGroups, global staff size 14
- `CONDUCTOR_SCORE_HEADER`: template for title, composer, arranger

**Part constants:**
- `PART_PAPER`: letter portrait `#(set-paper-size "letter")`, margins for readability
- `PART_LAYOUT`: measure numbers at every system start (`\override BarNumber.break-visibility = ##(#f #f #t)`), `barNumberVisibility = #first-bar-number-invisible`
- `PART_HEADER`: template for title, instrument name

**Studio mode variant:**
- `STUDIO_LAYOUT`: bar numbers on every measure (`barNumberVisibility = #all-bar-numbers-visible`, `\override BarNumber.break-visibility = ##(#t #t #t)`)

**Common:**
- `LILYPOND_VERSION`: `"2.24.4"` string
- `VERSION_HEADER`: `\\version "2.24.4"\n`

All constants are raw LilyPond text strings (not templates yet -- just the static blocks). The generator will interpolate instrument-specific content around these blocks.

No dedicated test file for stylesheet -- these are string constants verified transitively through score/part generator tests in Plan 04-02. The task is small enough to combine with Task 1's plan.
