**GREEN phase:** Create `src/engrave/rendering/generator.py`:

**`generate_music_definitions(music_vars: dict[str, str], global_music: str, chord_symbols: str | None = None) -> str`**
- Produces the `music-definitions.ly` content: version header, globalMusic variable, each instrument's concert-pitch music as a named variable, chord symbols if provided
- `music_vars` is a dict mapping variable_name -> LilyPond music content (concert pitch)
- Returns a complete LilyPond source string

**`generate_conductor_score(preset: BigBandPreset, music_var_names: list[str], has_chords: bool = True) -> str`**
- Produces the `score.ly` content: version header, `\include "music-definitions.ly"`, tabloid landscape paper, conductor score layout
- Builds StaffGroup hierarchy by iterating `preset.instruments` grouped by section:
  - For each section, create `\new StaffGroup = "{section}"` with appropriate systemStartDelimiter (bracket for winds, brace for rhythm)
  - Inside Rhythm section: `\new PianoStaff` for piano if `is_grand_staff`
  - `\new DrumStaff` for drums
- ChordNames context placed above top staff
- `\RemoveEmptyStaves` with first-system preservation
- `\midi { }` block for MIDI generation
- `\bookOutputName "score"`
- Staff size 14 for conductor score

**`generate_part(instrument: InstrumentSpec, preset_name: str = "Big Band", has_chords: bool = False, chord_var: str = "chordSymbols", studio_mode: bool = False) -> str`**
- Produces `part-{slug}.ly` content for one instrument
- `\include "music-definitions.ly"`
- Letter portrait paper
- If `instrument.is_transposing`: wrap music in `\transpose {from} {to}`
- Wrap in `\compressMMRests { << \globalMusic \{transpose_wrapper}\{variable_name} >> }`
- If `instrument.has_chord_symbols` and `has_chords`: add `\new ChordNames` context
- Measure number visibility at every system start
- Studio mode: bar numbers on every bar, no rehearsal letters
- `\bookOutputName "part-{slugified_name}"`
- `instrumentName` and `shortInstrumentName` in Staff \with block
- Staff size 20 (default)

Use `from engrave.rendering.ensemble import BigBandPreset, InstrumentSpec, StaffGroupType` and `from engrave.rendering.stylesheet import ...` for all layout constants.

Slug generation for part filenames: lowercase, spaces to hyphens, e.g., "Alto Sax 1" -> "alto-sax-1". Use a simple helper function (no python-slugify needed for instrument names -- they are ASCII).

Run: `uv run pytest tests/unit/test_score_generator.py tests/unit/test_part_generator.py -x` (all pass).

**GREEN phase:** Add `restate_dynamics(lilypond_source: str) -> str` to `generator.py`:
- Parse the LilyPond music text line by line / token by token
- Track the "current dynamic" state: updated whenever a dynamic marking (`\pp`, `\p`, `\mp`, `\mf`, `\f`, `\ff`, `\fff`, `\sfz`, `\fp`, `\sf`, `\spp`, `\sff`, etc.) is encountered
- Detect multi-bar rests: patterns like `R1*N` where N >= 2 (or consecutive `R1` bars totaling >= 2)
- After a rest of 2+ bars, at the next note entrance, insert the current dynamic if:
  1. A dynamic is currently tracked (there was a prior dynamic)
  2. The entrance does not already have a dynamic marking
- Return the modified LilyPond source string

Implementation approach: Use regex to find dynamic markings (`\\(pp|p|mp|mf|f|ff|fff|fp|sfz|sf|spp|sff)\b`) and multi-measure rests (`R[0-9]+(\*[0-9]+)?`). Walk through tokens tracking state. This is a text-processing pass -- it does NOT need to fully parse LilyPond, just recognize these specific patterns.

The function operates on a single instrument's music content (not the full .ly file). It will be called by the packager on each instrument's music before file generation, or integrated into generate_part's pipeline.

Run: `uv run pytest tests/unit/test_dynamic_restatement.py -x` (all pass).
