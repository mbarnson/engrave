       - `supported_formats: list[str] = ["mp3", "wav", "aiff", "flac"]`
       - Nested `SeparationConfig` model with:
         - `steps: list[SeparationStep] = []` where `SeparationStep` has `model: str`, `input_stem: str = "mix"`, `output_stems: list[str]`
         - `default_steps` class method returning big band default cascade (htdemucs_ft -> bs_roformer on other)
       - Nested `TranscriptionConfig` model with:
         - `backend: str = "basic_pitch"` (for future pluggability)
         - `venv_python: str | None = None` (path to Python 3.10 venv, None means in-process ONNX)
         - `onset_threshold: float = 0.5`
         - `frame_threshold: float = 0.3`
         - `minimum_note_length_ms: int = 58`
       - Nested `BenchmarkConfig` model with:
         - `soundfont_path: str | None = None`
         - `onset_tolerance: float = 0.05`
         - `results_dir: str = "data/benchmark_results"`
       - Add `audio: AudioConfig = AudioConfig()` to the `Settings` class.

    2. Create `src/engrave/audio/__init__.py` with docstring and empty `__all__` list (public API added in Plan 04).

    3. Create `src/engrave/audio/normalizer.py` with:
       - `normalize_audio(input_path: Path, output_path: Path, target_sr: int = 44100, channels: int = 1) -> Path`
         - Uses `pydub.AudioSegment.from_file()` for format detection
         - Converts to target sample rate and channel count
         - Exports as WAV
         - Validates file exists, raises `FileNotFoundError` if not
         - Validates format is supported (check extension), raises `ValueError` for unsupported
         - Validates duration does not exceed max_duration_seconds, raises `ValueError` if too long
         - Returns output_path
       - `detect_audio_format(input_path: Path) -> str` -- returns lowercase format string from file extension

       - Test format detection for supported formats
       - Test ValueError on unsupported format
       - Test FileNotFoundError on missing file
       - Test mono downmix (create stereo fixture, verify output is mono)
       - Mock pydub.AudioSegment for MP3/AIFF/FLAC tests (avoid needing actual encoded files)

    5. Update `tests/conftest.py` -- add a `sample_wav` fixture that creates a minimal valid WAV file (1 second, 44.1kHz, mono, sine wave at 440Hz) using the `wave` stdlib module. This fixture is reused across audio tests.

  <action>
    1. Create `src/engrave/audio/youtube.py` with:
       - `extract_youtube_audio(url: str, output_dir: Path) -> Path`
         - Uses `yt_dlp.YoutubeDL` Python API (NOT subprocess)
         - Sets `outtmpl` to `str(output_dir / '%(id)s.%(ext)s')` for deterministic filenames
         - Sets format to `bestaudio/best`
         - Adds `FFmpegExtractAudio` postprocessor with `preferredcodec='wav'`
         - Sets `quiet=True` to suppress console output
         - Extracts video ID from info dict
         - Returns `output_dir / f"{video_id}.wav"`
         - Raises `ValueError` for invalid/empty URLs
         - Wraps yt-dlp errors in a custom `YouTubeExtractionError` (subclass of RuntimeError) with clear message
       - `is_youtube_url(url: str) -> bool` -- validates URL matches YouTube patterns (youtube.com/watch, youtu.be, youtube.com/shorts)

    2. Create `tests/unit/test_youtube.py`:
       - Test `is_youtube_url()` with valid YouTube URLs (youtube.com/watch?v=xxx, youtu.be/xxx, youtube.com/shorts/xxx)
       - Test `is_youtube_url()` with non-YouTube URLs (returns False)
       - Test `extract_youtube_audio()` with MOCKED yt-dlp (patch `yt_dlp.YoutubeDL`) -- verify correct options are passed, verify output path construction from video ID
       - Test error handling: mock yt-dlp raising DownloadError, verify YouTubeExtractionError is raised
       - Do NOT actually download from YouTube in tests
