Source separation engine: hierarchical cascade with per-stem model routing.

Purpose: Separate mixed audio into individual instrument stems using audio-separator. The hierarchical cascade pattern enables progressive refinement -- first split into 4 stems (drums/bass/vocals/other), then further separate the "other" stem into piano/residual using dedicated models. This is critical for big band content where the "other" bin contains 11+ melodic instruments.

Output: `src/engrave/audio/separator.py` with hierarchical cascade logic, unit tests with mocked audio-separator.

    1. Create `src/engrave/audio/separator.py` with:

       - `@dataclass(frozen=True) class StemOutput:` -- stem_name (str), path (Path), model_used (str), step_index (int)

       - `def run_separation(audio_path: Path, steps: list[SeparationStep], output_dir: Path) -> list[StemOutput]:`
         - Validates audio_path exists
         - For each SeparationStep in order:
           a. Determine input file: if `step.input_stem == "mix"`, use original audio_path. Otherwise, find the output from a prior step matching `step.input_stem`.
           b. Create step output subdirectory: `output_dir / f"step-{step_index:02d}_{step.model.split('.')[0]}"`
           c. Instantiate `audio_separator.separator.Separator` with `output_dir=str(step_subdir)`, `output_format="WAV"`, `sample_rate=44100`
           d. Call `separator.load_model(model_filename=step.model)`
           e. Call `separator.separate(str(input_file))` -- returns list of output file paths
           f. Map output files to StemOutput objects using `_map_stem_names()` helper
           g. Collect all StemOutputs across all steps
         - Returns list of all StemOutput objects
         - IMPORTANT: Do NOT keep multiple Separator instances alive. Instantiate per-step, let GC reclaim. This prevents memory exhaustion on Apple Silicon.

       - `def _map_stem_names(output_files: list[str], expected_stems: list[str], model: str, step_index: int) -> list[StemOutput]:`
         - Maps audio-separator output filenames to the expected stem names from config
         - audio-separator naming varies by model architecture:
           - HTDemucs outputs: files containing "drums", "bass", "vocals", "other" (or "no_vocals"/"instrumental")
           - BS-RoFormer outputs: files containing "Vocals"/"Instrumental" or stem-specific names
         - Uses case-insensitive substring matching to associate output files with expected_stems
         - Falls back to positional assignment if name matching fails
         - Returns list of StemOutput with standardized stem names

       - `def get_default_steps() -> list[SeparationStep]:`
         - Returns the big band default cascade:
           1. SeparationStep(model="htdemucs_ft.yaml", input_stem="mix", output_stems=["drums", "bass", "vocals", "other"])
           2. SeparationStep(model="model_bs_roformer_ep_317_sdr_12.9755.ckpt", input_stem="other", output_stems=["piano", "residual"])
         - Note: Model filenames come from config. These defaults are overridable via engrave.toml.
