Transcription engine and quality annotation: pluggable MIDI transcription with post-transcription quality heuristics.

Purpose: Convert separated audio stems to MIDI via Basic Pitch with a protocol-based design enabling future backends. Quality metadata (note density, pitch range violations, onset clustering, velocity variance, duration distribution) travels downstream to inform the LLM about transcription confidence per stem -- this is the "LLM IS the fallback" strategy.

Output: `src/engrave/audio/transcriber.py` with protocol + Basic Pitch backend, `src/engrave/audio/quality.py` with 5 quality metrics, unit tests.
</objective>

       - `@dataclass class BasicPitchTranscriber:` implementing the Transcriber protocol:
         - Fields: `venv_python: Path | None = None` (None = in-process ONNX, Path = subprocess), `onset_threshold: float = 0.5`, `frame_threshold: float = 0.3`, `minimum_note_length_ms: int = 58`
         - `transcribe(self, wav_path: Path, output_dir: Path) -> Path:` -- dispatches to _transcribe_inprocess or _transcribe_subprocess based on venv_python
         - `_transcribe_inprocess(self, wav_path: Path, midi_path: Path) -> None:` -- imports basic_pitch.inference (predict, Model) and basic_pitch.ICASSP_2022_MODEL_PATH, creates Model, calls predict() with configured thresholds, writes MIDI via midi_data.write()
         - `_transcribe_subprocess(self, wav_path: Path, midi_path: Path) -> None:` -- runs `[str(self.venv_python), "-m", "basic_pitch", str(midi_path.parent), str(wav_path), "--model-serialization", "onnx"]` via subprocess.run with capture_output=True, text=True, timeout=300. Raises RuntimeError on non-zero returncode with stderr message.
         - Validates wav_path exists before transcription
         - Creates output_dir if it doesn't exist

       - `def create_transcriber(config: TranscriptionConfig) -> Transcriber:` -- factory function that reads config and returns a BasicPitchTranscriber with appropriate settings. If config.venv_python is set, uses subprocess path. Otherwise, uses in-process path.

<task type="auto">
  <name>Task 2: Post-transcription quality annotation</name>
  <files>
    src/engrave/audio/quality.py

  <action>
    1. Create `src/engrave/audio/quality.py` with:

       - `@dataclass class StemQuality:` with fields:
         - `stem_name: str`
         - `note_count: int`
         - `note_density_per_bar: float` -- notes per bar (too high = garbage, zero = separation ate the stem)
         - `pitch_range_violations: int` -- notes outside instrument's physical range
         - `onset_cluster_score: float` -- 0-1, high = suspicious simultaneous onsets on monophonic stem
         - `velocity_variance: float` -- standard deviation of velocities (low = suspicious flat dynamics)
         - `duration_cv: float` -- coefficient of variation of durations (low = model hallucinating a grid)
         - `to_dict(self) -> dict` -- serializes to dict for JSON storage in job metadata

       - `def annotate_quality(midi_path: Path, stem_name: str, tempo_bpm: float = 120.0, expected_range: tuple[int, int] = (0, 127)) -> StemQuality:`
         - Loads MIDI via pretty_midi.PrettyMIDI
         - Extracts all non-drum notes from all instruments
         - Returns zero-valued StemQuality if no notes found
         - Computes note_density_per_bar: len(notes) / estimated_bars (bars from duration_sec / 60 * tempo_bpm / 4)
         - Computes pitch_range_violations: count of notes outside expected_range
         - Computes onset_cluster_score: fraction of notes within 10ms of another note's onset
         - Computes velocity_variance: numpy std of note velocities
         - Computes duration_cv: numpy std(durations) / mean(durations)
         - Returns StemQuality

       - `INSTRUMENT_RANGES: dict[str, tuple[int, int]]` -- mapping from common instrument names to (lowest_midi_note, highest_midi_note). Include at minimum: piano (21, 108), bass (28, 67), guitar (40, 88), trumpet (55, 84), trombone (40, 72), alto_sax (49, 80), tenor_sax (44, 75), baritone_sax (36, 69), flute (60, 96), clarinet (50, 91), violin (55, 103), cello (36, 76), drums (35, 81).

       - `def get_expected_range(stem_name: str) -> tuple[int, int]:` -- looks up stem_name in INSTRUMENT_RANGES, returns (0, 127) if not found.

    2. Create `tests/unit/test_quality.py`:
       - Test annotate_quality with a programmatically created MIDI file (use pretty_midi to create a simple MIDI with known notes, write to tmp_path)
       - Test note_density_per_bar calculation with known note count and duration
       - Test pitch_range_violations: create MIDI with notes outside expected range, verify count
       - Test onset_cluster_score: create MIDI with simultaneous onsets, verify score > 0
       - Test velocity_variance: create MIDI with uniform velocity, verify low variance; create with varied velocity, verify higher variance
       - Test duration_cv: create MIDI with uniform durations, verify low cv; create with varied durations, verify higher cv
       - Test empty MIDI (no notes): verify zero-valued StemQuality returned
       - Test to_dict() serialization
       - Test get_expected_range() for known and unknown instruments
       - Test INSTRUMENT_RANGES includes the minimum set of instruments
  </action>
