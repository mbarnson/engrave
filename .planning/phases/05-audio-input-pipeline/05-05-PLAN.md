<objective>
Benchmark harness: closed-loop evaluation of the separation+transcription chain against ground truth MIDI.

Purpose: The experimentation harness is essential for empirically validating and comparing model choices. It renders corpus MIDI to audio (FluidSynth), runs that audio through the separation+transcription pipeline, and diffs the output MIDI against the original source MIDI using mir_eval. Results are stored as structured JSON for systematic comparison. Per user decision: this is pipeline-integrated, not a standalone tool.

Output: `src/engrave/benchmark/` package with renderer, evaluator, harness, data models, CLI `engrave benchmark` command, unit tests.
</objective>

       - `def render_midi_to_audio(midi_path: Path, wav_path: Path, soundfont: str | None = None) -> Path:`
         - Uses `midi2audio.FluidSynth(soundfont)` if soundfont provided, else `FluidSynth()` with default
         - Calls `fs.midi_to_audio(str(midi_path), str(wav_path))`

    4. Create `src/engrave/benchmark/evaluator.py` with:
       - `@dataclass class MidiDiffResult:` -- precision (float), recall (float), f1 (float), avg_overlap (float)
       - `def diff_midi(reference_path: Path, estimated_path: Path, onset_tolerance: float = 0.05) -> MidiDiffResult:`
         - Loads both MIDI files via pretty_midi.PrettyMIDI
         - Extracts (interval, pitch_hz) arrays from all non-drum instruments
         - Uses `mir_eval.transcription.precision_recall_f1_overlap()` with configurable onset_tolerance
         - Returns MidiDiffResult
         - Handles edge cases: empty reference (returns zeros), empty estimated (returns zeros), both empty (returns zeros with a note)
         - IMPORTANT: Convert both reference and estimated through pretty_midi to ensure consistent pitch/timing representation (per research pitfall #6)

    1. Create `src/engrave/benchmark/harness.py` with:

       - `class BenchmarkHarness:` with:
         - `__init__(self, pipeline: AudioPipeline, config: BenchmarkConfig)` -- stores pipeline and config
         - `run_single(self, midi_path: Path, results_dir: Path | None = None) -> BenchmarkRun:`
           a. Generate run_id from timestamp
           b. Create temp directory for intermediate artifacts
           c. **Render:** Call render_midi_to_audio(midi_path, temp_dir / "ground_truth.wav", config.soundfont_path)
           d. **Process:** Call pipeline.process(ground_truth_wav, temp_dir / "job") -- runs through separation + transcription
           e. **Evaluate:** For each stem MIDI in pipeline output, call diff_midi(midi_path, stem_midi_path, config.onset_tolerance)
              - Note: The reference MIDI is the FULL source, while estimated is per-stem. For a fair comparison, filter reference MIDI to matching instruments if possible, or compare aggregate. The simplest approach: compare each stem's MIDI against the full reference and note that recall will be low for individual stems (each stem captures only part of the full score). This is expected -- the meaningful metric is whether the stem's notes are CORRECT (precision), not whether it captured everything (recall).
           f. Compute AggregateScore across all stems
           g. Build and return BenchmarkRun
           h. If results_dir provided, save BenchmarkRun to JSON

         - `run_batch(self, midi_paths: list[Path], results_dir: Path) -> list[BenchmarkRun]:`
           - Runs run_single for each MIDI path
           - Returns list of BenchmarkRun
           - Saves all to results_dir

         - `@staticmethod compare_runs(run_paths: list[Path]) -> str:`
           - Loads BenchmarkRun objects from JSON files
           - Formats comparison table (model config, aggregate F1, per-stem F1)
           - Returns formatted string for CLI output
