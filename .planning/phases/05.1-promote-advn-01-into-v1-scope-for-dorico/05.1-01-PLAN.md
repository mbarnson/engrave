  <behavior>
    **Pitch Map (pitch_map.py):**
    - ly_pitch_to_m21("bf4") -> "B-4"
    - ly_pitch_to_m21("fis5") -> "F#5"
    - ly_pitch_to_m21("c3") -> "C3"
    - ly_pitch_to_m21("ees3") -> "E-3"
    - ly_pitch_to_m21("invalid") -> raises ValueError
    - ly_key_to_m21("bf_major") -> "B-"
    - ly_key_to_m21("fs_minor") -> "f#"
    - Covers all standard LilyPond accidental variants: f/s, es/is, ef/bf vs ees/bes

    **Pydantic Models (models.py):**
    - NoteEvent: pitch (str|None), type (str|None), beat (float), duration (float), articulations (list[str]|None), expressions (list[str]|None), dynamic (str|None)
    - MeasureData: number (int), notes (list[NoteEvent])
    - SectionNotation: instrument (str), key (str|None), time_signature (str|None), measures (list[MeasureData])
    - Valid JSON parses without error
    - Missing required fields (beat, duration) raise ValidationError
    - Invalid pitch format raises ValidationError
    - "type": "rest" with no pitch is valid
    - Pitched note with no "type" field is valid

    **Builder (builder.py):**
    - build_note(NoteEvent(pitch="bf4", beat=1.0, duration=1.0)) -> music21.note.Note with pitch B-4, quarterLength 1.0
    - build_note(NoteEvent(type="rest", beat=1.0, duration=4.0)) -> music21.note.Rest with quarterLength 4.0
    - build_note with articulations=["accent", "staccato"] -> Note with Accent and Staccato articulations
    - build_note with dynamic="f" -> Note with Dynamic("f") attached
    - build_note with expressions=["fermata"] -> Note with Fermata expression
    - build_measure(MeasureData) -> music21.stream.Measure with correct number and notes
    - build_part(instrument, sections_json, key, time_sig, tempo) -> music21.stream.Part with correct hierarchy
    - build_score(all_sections_json, instruments, key, time_sig, tempo) -> music21.stream.Score
    - ARTICULATION_MAP dict maps LilyPond articulation names to music21 classes
    - EXPRESSION_MAP dict maps expression names to music21 classes
  </behavior>
  <implementation>
    **pitch_map.py:**
    - Static _LY_NOTE_TO_M21 dict covering c-b, flat variants (f/ef/cf/etc + es/ees/ces/etc), sharp variants (s/is)
    - _LY_PITCH_RE regex: ^([a-g](?:es|is|f|s|ees|bes|ais|...)?)(\d)$ -- order accidental patterns longest-first to prevent partial matches
    - ly_pitch_to_m21(ly_pitch: str) -> str: parse regex, lookup note name, append octave
    - ly_key_to_m21(ly_key: str) -> str: split on underscore, convert note name via _LY_NOTE_TO_M21, return music21 key string (uppercase for major, lowercase for minor)

    **models.py:**
    - Pydantic BaseModel subclasses with field validators
    - NoteEvent: @field_validator("pitch") validates against LilyPond pitch regex (allow None for rests)
    - NoteEvent: @model_validator validates that type="rest" notes have no pitch, and pitched notes have no type="rest"
    - SectionNotation: top-level model for one instrument's section output

    **builder.py:**
    - ARTICULATION_MAP: accent->Accent, marcato->StrongAccent, staccato->Staccato, staccatissimo->Staccatissimo, tenuto->Tenuto, portato->DetachedLegato, doit->Doit, falloff->Falloff, scoop->Scoop, plop->Plop
    - EXPRESSION_MAP: fermata->Fermata, trill->Trill, turn->Turn, mordent->Mordent
    - build_note: create Note or Rest, set quarterLength, attach articulations/expressions/dynamics
    - build_measure: create Measure with number, append notes
    - build_part: create Part with partName, insert KeySignature + TimeSignature + MetronomeMark in first measure, append all measures
    - build_score: create Score, build one Part per instrument from collected section data, return Score
    - Use ly_pitch_to_m21 for pitch conversion, ly_key_to_m21 for key conversion
    - Log warnings for unmapped articulation/expression names (don't crash)

    **__init__.py:**
    - Re-export public API: NoteEvent, MeasureData, SectionNotation, ly_pitch_to_m21, ly_key_to_m21, build_note, build_measure, build_part, build_score, ARTICULATION_MAP, EXPRESSION_MAP
  </implementation>
</feature>
