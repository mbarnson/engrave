## Accomplishments

- Pipeline fans out two concurrent requests per section via asyncio.gather (Chatterfart prefix-caching pattern)

## Decisions Made
- JSON extraction uses three-stage fallback: parse as array, parse as object and wrap, regex extraction of individual objects -- never raises
- _request_json_notation is isolated in a separate coroutine so any exception (network, parse, timeout) is caught without affecting the LilyPond coroutine running in parallel
- asyncio.gather catches NotImplementedError to fall back to sequential dispatch for routers that don't support concurrent calls
- Training pairs saved as section_{N}.json with both ly_source and json_notation fields for TUNE-01/TUNE-02 fine-tuning data
