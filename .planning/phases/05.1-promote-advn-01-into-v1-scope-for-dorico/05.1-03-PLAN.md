  <action>
**validator.py:**
- Create `validate_musicxml(musicxml_path: Path, xsd_path: Path | None = None) -> tuple[bool, str]`
- Use `xmlschema.XMLSchema(str(xsd_path)).validate(str(musicxml_path))` per RESEARCH.md
- Default xsd_path to vendored schema at `src/engrave/musicxml/schema/musicxml.xsd`
- Return (True, "") on valid, (False, error_message) on invalid
- Handle xmlschema.XMLSchemaValidationError gracefully

**Vendor MusicXML 4.0 XSD files:**
- Download MusicXML 4.0 XSD files (musicxml.xsd, common.xsd, container.xsd, sounds.xsd, etc.) from the W3C GitHub repository
- Place in `src/engrave/musicxml/schema/` directory
- These are small, stable files required for offline validation per RESEARCH.md open question #4

**telemetry.py:**
- Create `log_disagreement(section_index: int, instrument: str, ly_source: str, json_data: dict, reason: str, output_dir: Path | None = None) -> None`
- Log to Python logger at WARNING level with structured message
- If output_dir provided, also write JSON file to `{output_dir}/telemetry/disagreement_{section}_{instrument}.json` containing all parameters plus timestamp
- This feeds TUNE-02 fine-tuning data per CONTEXT.md decision

**json_assembler.py:**
- Create `assemble_musicxml(json_sections: list[list[dict] | None], instrument_names: list[str], key_sig: str, time_sig: str, tempo_bpm: int, output_path: Path, xsd_path: Path | None = None) -> tuple[bool, Path | None]`
- Filter out None sections (where JSON generation failed)
- For each instrument, collect measures across all sections
- Use Pydantic `SectionNotation.model_validate()` to validate each section's JSON
- Use `build_score()` from musicxml/builder.py to construct music21 Score
- Write MusicXML via `score.write("musicxml", fp=str(output_path))`
- Validate via `validate_musicxml(output_path, xsd_path)`
- Return (True, output_path) on success, (False, None) on failure
- Log warnings for validation failures but never raise -- graceful degradation per CONTEXT.md
- Concert pitch throughout -- no transposition applied (matches Engrave's internal storage convention per CONTEXT.md)

**Tests:**
- test_xsd_validator.py: test valid MusicXML passes, test invalid XML fails with error message, test missing file handled gracefully
- test_telemetry.py: test log_disagreement writes to logger, test file output when output_dir provided, test no crash when output_dir is None
  </action>
