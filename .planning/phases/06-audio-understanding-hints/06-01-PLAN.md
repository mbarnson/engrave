**1. Create `src/engrave/audio/description.py`** -- Pydantic models for the two-tier audio description schema.

Per user decision (locked):
- `SectionDescription(BaseModel)`: label (str, default ""), start_bar (int, default 1), end_bar (int, default 1), key (str | None, default None -- None if no change), active_instruments (list[str], default_factory=list), texture (str, default "" -- describes musical texture like "walking bass under trumpet melody"), dynamics (str, default "" -- e.g. "mf", "ff"), notes (str | None, default None -- nullable catch-all for stray observations, separate from texture).
- `AudioDescription(BaseModel)`: Track-level: tempo_bpm (int, default 120), tempo_variable (bool, default False), time_signature (str, default "4/4"), key (str, default "C major" -- root + mode), instruments (list[str], default_factory=list), style_tags (list[str], default_factory=list), energy_arc (str, default ""). Per-section: sections (list[SectionDescription], default_factory=list).
- No confidence scores (locked decision).

**2. Create `src/engrave/audio/describer.py`** -- Describer protocol and GeminiDescriber.

Mirror the Transcriber protocol pattern from `src/engrave/audio/transcriber.py`:
- `@runtime_checkable class Describer(Protocol)`: async method `describe(audio_path: Path, sections: list[SectionBoundary], tempo_bpm: int, ticks_per_beat: int) -> AudioDescription`. Import SectionBoundary from `engrave.midi.sections`.
- `class GeminiDescriber`: Implements Describer. Constructor takes `model: str = "gemini/gemini-3-flash"`, `timeout: int = 120`.
  - `describe()` method: (a) Build the analysis prompt listing the section boundaries by bar range, asking the audio LM to identify tempo, time sig, key, instruments, style tags, energy arc at track level, and per-section: active instruments, texture, dynamics, notes. Emphasize: "Focus on musical structure and character, NOT individual note accuracy." (b) Build LiteLLM-compatible messages with base64-encoded audio as `type: "file"` with `file_data: "data:audio/wav;base64,{encoded}"`. (c) Call `litellm.acompletion()` with model, messages, `response_format={"type": "json_schema", "json_schema": {"name": "AudioDescription", "schema": AudioDescription.model_json_schema()}}`, temperature=0.2. (d) Parse response via `AudioDescription.model_validate_json()`. (e) On validation error, retry once with a simplified prompt ("Return valid JSON matching the schema"). On second failure, return a minimal AudioDescription with defaults. Log warnings on fallback.
  - Audio file size check: if file > 15MB, downsample to 16kHz before base64 encoding (per research pitfall #2). Use pydub AudioSegment for downsampling since it's already a project dependency.
- `def create_describer(config) -> Describer`: Factory mirroring `create_transcriber()` pattern.

**3. Create `src/engrave/audio/templates.py`** -- Natural language template rendering.

Per user decision: audio description injected as natural language summary, not raw JSON.
- `render_section_description(section: SectionDescription) -> str`: Build readable sentence from non-empty fields. Pattern: "Section: {label}. Key: {key}. Active instruments: {instruments}. Texture: {texture}. Dynamics: {dynamics}. Notes: {notes}." Only include non-empty/non-None fields.
- `render_track_description(desc: AudioDescription) -> str`: Build global header summarizing track-level metadata: "Tempo: {tempo_bpm} BPM{' (variable)' if tempo_variable}. Time signature: {time_sig}. Key: {key}. Style: {style_tags joined}. Instruments: {instruments joined}. Energy arc: {energy_arc}." Only include non-empty fields.
- `render_full_description(desc: AudioDescription) -> str`: Combine track description + per-section descriptions into a single block. Track header first, then each section on a new line.

**4. Extend `src/engrave/config/settings.py`** -- Add DescriberConfig.

- `class DescriberConfig(BaseModel)`: model (str, default "gemini/gemini-3-flash"), timeout (int, default 120), max_inline_size_mb (int, default 15), downsample_rate (int, default 16000).
- Add `describer: DescriberConfig = DescriberConfig()` field to `AudioConfig`.
