    - `SectionDescription(BaseModel)` with fields per user decision:
      - `label: str = ""` (e.g., "intro", "verse-1", "chorus-1")
      - `start_bar: int = 1`
      - `end_bar: int = 1`
      - `key: str | None = None` (e.g., "Bb major" -- None if no change from track level)
      - `active_instruments: list[str] = Field(default_factory=list)`
      - `texture: str = ""` (e.g., "solo piano", "full ensemble block chords")
      - `dynamics: str = ""` (e.g., "mf", "ff", "building from mp to f")
      - `notes: str | None = None` (nullable catch-all for stray observations, SEPARATE from texture per user decision)
    - `AudioDescription(BaseModel)` with track-level fields per user decision:
      - `tempo_bpm: int = 120`
      - `tempo_variable: bool = False`
      - `time_signature: str = "4/4"`
      - `key: str = "C major"` (root + mode)
      - `instruments: list[str] = Field(default_factory=list)`
      - `style_tags: list[str] = Field(default_factory=list)` (e.g., ["swing", "big band", "blues"])
      - `energy_arc: str = ""` (e.g., "mp -> mf -> f -> ff -> mf")
      - `sections: list[SectionDescription] = Field(default_factory=list)`
    - NO confidence scores (explicitly rejected per user decision)

    **2. `src/engrave/audio/describer.py` -- Protocol + GeminiDescriber + factory**
    - `Describer` Protocol (runtime_checkable) with async method:
      ```python
      async def describe(self, audio_path: Path, sections: list[SectionBoundary],
                         tempo_bpm: int, ticks_per_beat: int) -> AudioDescription
      ```
      Key difference from Transcriber: Describer is async (per user decision -- I/O-bound audio LM calls)
    - `GeminiDescriber` class:
      - Constructor takes `model: str`, `api_key: str | None`, `timeout: int`, `max_file_size_mb: int`
      - `describe()` method:
        1. Build prompt from sections (bar ranges, request for structured analysis focusing on musical structure/character, NOT note accuracy per user decision)
        2. Build messages with base64-encoded audio content (WAV). If file > `max_file_size_mb * 1024 * 1024`, downsample to 16kHz first using pydub before encoding (per pitfall #2 in research -- Gemini downsamples internally anyway)
        3. Call `litellm.acompletion()` with `model=self.model`, `response_format={"type": "json_schema", "json_schema": {"name": "AudioDescription", "schema": AudioDescription.model_json_schema()}}`, `temperature=0.2`
        4. Parse response via `AudioDescription.model_validate_json(response.choices[0].message.content)`
        5. On validation error: retry once with simplified prompt (max 1 retry). On second failure, return minimal AudioDescription with only defaults and log warning
        6. On timeout/API error: log warning, return minimal AudioDescription with defaults (per Claude's discretion: skip with warning rather than crash)
    - Describer bypasses InferenceRouter (per user decision -- different input modality, different API surface)
    - `create_describer(config: DescriberConfig) -> Describer` factory function mirroring `create_transcriber` pattern

    **3. `src/engrave/audio/templates.py` -- NL template rendering**
    - `render_track_summary(desc: AudioDescription) -> str`: Converts track-level metadata to a compact natural language header sentence. Example output: "Swing big band in Bb major, 142 BPM, 4/4 time. Instruments: trumpet, trombone, alto sax, piano, bass, drums. Energy: mp -> mf -> f -> ff -> mf."
    - `render_section_description(section: SectionDescription) -> str`: Converts a single section annotation to a natural language sentence. Example: "Section: verse-1 (bars 9-24). Key: Bb major. Active instruments: trumpet, piano, bass, drums. Texture: walking bass under trumpet melody. Dynamics: mf."
      - Only include fields that have non-empty/non-None values (skip empty fields)
      - Include notes field if non-None: "Notes: sounds like a Basie arrangement."
    - `render_full_description(desc: AudioDescription) -> str`: Combines track summary + all section descriptions into a multi-line string for prompt injection. This is what gets injected into the CONTEXTUAL block of the three-tier prompt

    **4. `src/engrave/audio/__init__.py` -- Update exports**
    - Add `AudioDescription`, `SectionDescription` from description module
    - Add `Describer`, `GeminiDescriber`, `create_describer` from describer module
    - Add `render_full_description`, `render_track_summary`, `render_section_description` from templates module

    **5. `src/engrave/config/settings.py` -- Add DescriberConfig**
    - New `DescriberConfig(BaseModel)` class:
      - `model: str = "gemini/gemini-3-flash"` (LiteLLM model prefix)
      - `timeout: int = 120` (seconds)
      - `max_file_size_mb: int = 15` (threshold for downsampling before inline base64)
      - `max_retries: int = 1` (retries on validation failure)
    - Add `describer: DescriberConfig = DescriberConfig()` field to `AudioConfig`
    - This nests under `[audio.describer]` in engrave.toml
