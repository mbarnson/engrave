    """Load hints from inline text or file path.

    If raw is an existing file path, reads and returns file content.

**3. Create `src/engrave/generation/audit.py`** -- Structured audit log.

Per user decision (locked): structured JSON, per-field override tracking. Fields: {field, midi_value, audio_value, hint_value, resolved_to, source}.

- `@dataclass class FieldResolution`: field (str), midi_value (str | None), audio_value (str | None), hint_value (str | None), resolved_to (str), source (str -- "hint", "audio", or "midi").
- `@dataclass class AuditEntry`: section_index (int), section_label (str), timestamp (str, default ""), resolutions (list[FieldResolution], default_factory=list).
- `@dataclass class AuditLog`: job_id (str, default ""), entries (list[AuditEntry], default_factory=list).
  - `write(output_dir: Path) -> Path`: Write audit log as JSON file (`audit_log.json`) in output_dir. Uses `json.dumps(asdict(self), indent=2)`.
  - `add_entry(entry: AuditEntry)`: Append entry to entries list.

Use dataclasses (not Pydantic) since this is output-only data, not validated input. Import from `dataclasses` and `json`.

**4. Modify `src/engrave/generation/prompts.py`** -- Restructure to three-tier authority template.

Per user decisions (locked):
- Same three-tier template always, even for pure MIDI input (CONTEXTUAL section empty/absent when no audio).
- Three-tier priority hardcoded: (1) DEFINITIVE = user hints, (2) CONTEXTUAL = audio analysis, (3) RAW INPUT = MIDI.
- Audio description injected as natural language summary, not raw JSON.
- Prompt structure: compact global header + per-section audio description summary + user hints (full block) + current section MIDI data + coherence state.

Modify `build_section_prompt()` signature to add two new optional parameters:
```python
def build_section_prompt(
    section_midi: dict[str, str],
    coherence: CoherenceState,
    rag_examples: list[str],
    template: str,
    budget: PromptBudget | None = None,
    audio_description: str = "",    # NEW: rendered NL summary from templates.py
    user_hints: str = "",           # NEW: raw user hint text
) -> str:
```

Keep backward compatibility: existing callers without audio_description/user_hints still work (empty defaults produce placeholder text).

Add `description_tokens: int = 800` to PromptBudget. Reduce `safety_margin` from 4000 to 3200 to accommodate (per research pitfall #3). Audio description and user hints are never truncated (they are small and high-authority).

Restructure the prompt body to use three-tier format:
```
=== DEFINITIVE (User Hints -- always authoritative) ===
{user_hints or "No user hints provided."}

=== CONTEXTUAL (Audio Analysis -- structural observations) ===
{audio_description or "No audio analysis available."}

=== CURRENT MUSICAL STATE ===
{coherence_text}

LILYPOND TEMPLATE (fill in the instrument variables):
{template}

SIMILAR EXAMPLES FROM CORPUS:
{rag_text}

=== RAW INPUT (MIDI Transcription -- treat as noisy suggestion) ===
{midi_text}
```

Keep all existing RULES at the top unchanged. The budget fitting logic remains unchanged for midi, rag, and coherence components. Audio description and user hints do NOT go through budget fitting (per research: they are small and never truncated).
