<objective>
Wire user hints and audio descriptions into the generation pipeline: hint loading, three-tier prompt restructuring, audit log, pipeline integration, and CLI extension.

Purpose: AUDP-04 requires natural language hints with authoritative override. This plan restructures the existing generation prompt to a three-tier authority format (DEFINITIVE/CONTEXTUAL/RAW INPUT), builds the hint loading system, adds the audit log for per-field source tracking, integrates both audio descriptions and hints into the pipeline, and extends the CLI with `--hints`.

Output: Hint loader, restructured prompts, audit log, pipeline integration with audio description + hints, CLI `--hints`, and integration tests covering both audio and MIDI-only paths.
</objective>

      - If raw is None, return ""

    - Per user decision: free text, no DSL, no regex extraction, LLM is the parser

    **3. `src/engrave/generation/audit.py` -- Audit log**
    - `FieldResolution` dataclass: `field: str`, `midi_value: str | None = None`, `audio_value: str | None = None`, `hint_value: str | None = None`, `resolved_to: str = ""`, `source: str = ""` (one of "hint", "audio", "midi")
    - `AuditEntry` dataclass: `section_index: int`, `section_label: str`, `timestamp: str = ""`, `resolutions: list[FieldResolution] = field(default_factory=list)`
    - `AuditLog` dataclass: `job_id: str = ""`, `entries: list[AuditEntry] = field(default_factory=list)`
    - `AuditLog.add_entry(section_index, section_label, resolutions) -> None`: Creates timestamped AuditEntry and appends
    - `AuditLog.write(output_dir: Path) -> Path`: Writes `audit_log.json` to output_dir using `json.dumps(asdict(self), indent=2)`, returns path
    - Per user decision: audit log is skeletal in Phase 6 -- hint_value is always None (hints are unstructured text), audio_value requires parsing NL. Infrastructure is forward-looking
    - Per user decision: structured JSON, machine-readable, queryable via jq

          coherence: CoherenceState,
          rag_examples: list[str],

      {template}

      {rag_text}

    - The RULES section stays unchanged
    - Preserve backward compatibility: existing callers omitting the new parameters get the same prompt structure with empty DEFINITIVE and CONTEXTUAL sections
