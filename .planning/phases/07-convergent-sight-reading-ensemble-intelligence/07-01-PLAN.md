    ## Component 1: Token Scanner (ENSM-03)

    `apply_articulation_defaults(ly_source: str) -> tuple[str, list[dict]]`

    Returns modified LilyPond source and a list of resolution telemetry records (for staccato+accent cases).

    Rules (per user decision -- purely syntactic, by duration only):
    1. Unmarked quarter notes (duration `4`) get staccato (`-.`): `c'4` -> `c'4-.`
    2. Unmarked eighth notes (duration `8`) stay unmarked (long default): `d'8` -> `d'8` (no change)
    3. Staccato+accent (`-.->` or combination) resolves to accent only: `e'4-.->` -> `e'4->`. Log each resolution as telemetry dict: `{"bar": N, "beat": B, "original": "-.->", "resolved": "->"}`
    4. Swing assumed unless "Straight 8s" -- handled by beaming (ENGR-05), not this function

    Implementation: Token-walking regex scanner modeled on `restate_dynamics()` in `generator.py`. Use position-based scanning with `re.match(pattern, source, pos)`.

    Critical parsing details (from RESEARCH.md pitfalls):
    - Regex must capture pitch, duration, and articulation as separate groups
    - Dotted duration `c4.` (dot attached to duration) vs staccato `c4-.` (shorthand) -- duration group `(\d+\.?)` captures the dot, articulation group is separate
    - Sticky durations: track "current duration" state. When a note lacks explicit duration, apply the most recent duration for rule matching
    - Rests (`r4`, `r8`) should pass through unchanged -- no articulation defaults on rests
    - Only modify unmarked notes -- if a note already has any articulation mark, skip it
    - Articulation shorthand patterns: `-. -> -- -^ -! -_` (anchored immediately after duration/dot)

    Cases:
    - `c'4` -> `c'4-.` (quarter gets staccato)
    - `d'8` -> `d'8` (eighth stays unmarked)
    - `e'4-.` -> `e'4-.` (already has staccato, no change)
    - `f'4->` -> `f'4->` (already has accent, no change)
    - `g'4-.->` -> `g'4->` (staccato+accent resolves to accent)
    - `a'4-.-^` -> `a'4-^` (staccato+marcato: keep marcato, strip staccato per accent-subsumes logic... wait, user decision says staccato+accent specifically. For staccato+marcato, keep both? User says "staccato+accent never paired" -- accent subsumes. Marcato is different. Keep staccato+marcato as-is)
    - `c'4.` -> `c'4.-.` (dotted quarter gets staccato -- it's still a quarter-note family)
    - `r4` -> `r4` (rests unchanged)
    - `c' d' e'` (sticky duration from earlier `c'4`) -> apply rule based on tracked duration

    ## Component 2: Rhythmic Aligner (ENSM-05)

    `apply_section_consistency(part_sources: dict[str, str], time_sig: tuple[int, int] = (4, 4)) -> dict[str, str]`

    Takes dict of `{variable_name: ly_content}` for all parts in a section group. Returns modified dict with redundant articulations stripped.

    Algorithm:
    1. Build beat map for each part: `dict[tuple[int, float], BeatEvent]` mapping (bar, beat) to articulation marks
    2. For each (bar, beat) coordinate:
       a. Collect articulations from all sounding parts (skip rests via `is_rest` flag)
       b. If fewer than 2 sounding parts at this position, skip (nothing to compare)
       c. If all sounding parts have identical articulation marks -> strip eligible marks from all
       d. If any differ -> keep all markings unchanged
    3. Eligible marks for omission: staccato, accent, tenuto, marcato (explicit allowlist per user decision)
    4. Dynamics, tempo markings, expression text, rehearsal marks: NEVER stripped

    BeatEvent dataclass:
    ```python
    @dataclass
    class BeatEvent:
        bar: int
        beat: float
        articulations: list[str]  # ["-.", "->", etc.]
        is_rest: bool
    ```

    Duration-to-beats lookup: `{1: 4.0, 2: 2.0, 4: 1.0, 8: 0.5, 16: 0.25, 32: 0.125}` with dotted multiplier (1.5x).

    Bar boundary tracking: accumulate beats, when total >= beats_per_bar, increment bar number.

    Cases:
    - All 4 trumpets have `-.` on beat 1 bar 1 -> strip `-.` from all
    - Trumpets 1-3 have `-.`, trumpet 4 has `->` on same beat -> keep all markings
    - Trumpets 1-3 have `-.`, trumpet 4 is a rest -> strip `-.` from 1-3 (rest excluded)
    - All trumpets have `\f` on same beat -> keep `\f` on all (dynamics never stripped)
    - Mixed articulation+dynamics: `-.` same on all, `\f` on all -> strip `-.`, keep `\f`
  </behavior>
  <implementation>
    Follow the token-walking regex pattern from `restate_dynamics()`:
    1. Define compiled regex patterns at module level
    2. Walk source with pos-based scanning
    3. Build result parts list
    4. Join at end

    For the rhythmic aligner, build beat maps first, then do a second pass stripping matched articulations using string replacement on the original source.

    Processing pipeline order (per user decision): LLM output -> apply_articulation_defaults() -> apply_section_consistency() -> final LilyPond
  </implementation>
