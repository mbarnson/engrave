Purpose: This is the core differentiator -- joint generation enables convergent sight-reading where musicians in a section independently arrive at the same interpretation. The pipeline restructuring reduces LLM calls from 17 to 7 per section while producing musically coherent section parts.

  <action>
    Restructure `generate_from_midi()` in `pipeline.py` to dispatch per section-group instead of per instrument. The core changes:

    **1. Import new modules at top of pipeline.py:**
    ```python
    from engrave.generation.section_groups import resolve_section_groups, beaming_commands, resolve_beam_style, BeamStyle
    from engrave.rendering.articulation import apply_articulation_defaults, apply_section_consistency
    ```

    **2. In `generate_from_midi()`, after building instrument_names (step 3), resolve section groups:**
    ```python
    from engrave.rendering.ensemble import BIG_BAND  # or accept preset as parameter
    groups = resolve_section_groups(preset.instruments)
    ```

    The function should accept an optional `preset` parameter (default: BIG_BAND). When no preset is provided, fall back to per-instrument generation (each instrument is its own group).

    **3. Replace the per-instrument section loop with per-group dispatch:**

    For each temporal section (bar range):
      a. Resolve `beam_style` for this section (using audio description from Phase 6 if available, defaulting to SWING)
      b. For each section group:
         - Filter MIDI tracks to only the instruments in this group
         - Tokenize only those tracks' notes for the section bar range
         - Build a group-scoped prompt containing:
           * MIDI tokens for group instruments only
           * RAG examples filtered/queried for group instruments (e.g., "trumpet section swing")
           * Coherence state for this group (from previous temporal section's group output)
           * Template with variable placeholders for group instruments only
           * Beaming commands for the resolved beam_style injected into the template
         - Dispatch `generate_section()` with group-scoped parameters
         - The existing `parse_instrument_blocks()` handles multi-instrument LLM output (no change needed)
         - Missing instruments default to `R1` (existing fallback in pipeline.py line 263)
      c. After all groups for this temporal section complete:
         - Apply `apply_articulation_defaults()` to each part's LilyPond content
         - For each section group with 2+ instruments: apply `apply_section_consistency()` on the group's parts
         - Log any staccato+accent resolution telemetry to the audit log (same pattern as Phase 6 failure_log.py)
      d. Collect processed part sources for assembly

    **4. Maintain per-group coherence state:**
    Change from single `CoherenceState` to `dict[str, CoherenceState]` keyed by group identifier (section_group name for grouped instruments, variable_name for ungrouped). Initialize each from the analysis (same as current). Update each from its own group's output after each temporal section.

    **5. Update `build_section_prompt()` in prompts.py** to accept a list of instrument names (scoped to group) instead of all instruments. The existing function signature may already work if it takes `section_midi: dict[str, str]` which is naturally scoped. Verify and adjust if needed.

    **6. Update `build_score_template()` in templates.py** to accept an optional `beaming: str = ""` parameter that injects beaming commands at the beginning of the score template when non-empty.

    **7. JSON fan-out adjustment:**
    For section groups, the JSON notation request should produce one JSON blob containing all instruments in the group (array of `{"instrument": "trumpet_1", "measures": [...]}` objects per user decision). Update `_request_json_notation()` to pass group instrument names. The existing `build_json_generation_suffix()` already takes instrument_names and should work with a subset.

    **8. Error handling:**
    If a section-group generation call fails (LLM error, compilation failure), fail the entire temporal section (matching existing behavior from 03-03 decision: "section failure halts entire generation"). Log which group failed for debugging.

    **Key constraints (from user decisions):**
    - No cross-section-group coherence sharing within the same bar range (preserves parallelism)
    - Existing `parse_instrument_blocks()` delimiter pattern unchanged
    - Processing order: LLM output -> ENSM-03 defaults -> ENSM-05 omission -> final LilyPond
    - Section-sequential coherence only (group gets state from its own previous temporal section)
