User reviewed the PDF and identified 3 problems:

## 1. Bass in treble clef
The assembler `\score` block uses bare `\new Staff` with no clef.
Fix location: assembler.py staff_lines builder (line ~136).
Need: lookup clef from BigBandPreset or MIDI track info.
Bass -> \clef bass, Trombones -> \clef bass, Drums -> \clef percussion.

## 2. Paper size / layout cutoff
The standalone assembled .ly has NO \paper block.
LilyPond defaults to letter portrait, content is cut off on right side.
Fix: assembler should include CONDUCTOR_SCORE_PAPER from stylesheet.py
(tabloid landscape: `#(set-paper-size "tabloid" 'landscape)`)
Also needs CONDUCTOR_SCORE_LAYOUT for staff sizing.

## 3. Too many dynamics (ff/fff on every note)
MIDI transcription converts per-note velocity to dynamics.
Music notation only uses dynamics when the level CHANGES.
Fix location: likely in MIDI transcriber or in a post-processing step.
Check: src/engrave/midi/ for velocity-to-dynamics logic.
Also check: restate_dynamics() in rendering/packager.py (line 167) --
  this may already exist but isn't being called in the standalone path.
The rendering pipeline has `restate_dynamics()` but the standalone compile
bypasses it entirely.
</quality_issues>
