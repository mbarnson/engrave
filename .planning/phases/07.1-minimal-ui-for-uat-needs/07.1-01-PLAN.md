  <action>
Create the `src/engrave/smoke/` package with three modules:

**`__init__.py`**: Re-export `run_smoke_test`, `SmokeResult`, `InputResult`, `CheckResult`.

**`runner.py`**: Implement the smoke test runner:
- Data classes: `CheckResult(name, passed, message, details)`, `InputResult(input_path, pipeline_path, checks, elapsed_seconds, error)`, `SmokeResult(inputs, total_passed, total_failed, total_errors, elapsed_seconds)`.
- `AUDIO_EXTENSIONS = {".wav", ".mp3", ".flac", ".aiff"}` and `MIDI_EXTENSIONS = {".mid", ".midi"}`.
- `discover_inputs(test_dir: Path) -> list[tuple[Path, str]]`: Iterate `sorted(test_dir.iterdir())`, match extensions, return `(path, "audio"|"midi")` tuples.
- `async run_smoke_test(test_dir: Path) -> SmokeResult`: For each discovered input, run the pipeline (audio path uses `AudioPipeline.process()` then `generate_from_midi()` on transcribed stems then `RenderPipeline.render()`; MIDI path uses `generate_from_midi()` then `RenderPipeline.render()` directly). Catch any exception as input-level error. After pipeline completes (or errors), run all checks via `run_checks()`. Collect timing. Aggregate totals.
- Before running any inputs, check LLM provider connectivity by attempting to instantiate `Settings()` and `InferenceRouter(settings)` and calling `router.complete(role="generator", ...)` with a trivial prompt. If this fails, report clearly and return early (per Pitfall 3 from RESEARCH.md).
- Use `from engrave.rendering.ensemble import BIG_BAND` to determine expected instrument count for checks 5 and 9.

**`checks.py`**: Implement 9 check functions, each returning `CheckResult`:
1. `check_no_exceptions(error: str | None)` -- Pass if error is None.
2. `check_compilable_ly(zip_path: Path)` -- Pass if all .ly files in ZIP have corresponding .pdf files (do NOT re-compile; the pipeline already compiled them. Check that for each .ly file except music-definitions.ly, a .pdf with matching stem exists).
3. `check_valid_pdfs(zip_path: Path)` -- Pass if PDF files exist and are non-empty.
4. `check_valid_musicxml(zip_path: Path)` -- Extract .musicxml from ZIP. If no .musicxml exists, return CheckResult with passed=True, name="valid_musicxml", message="SKIPPED (MusicXML not generated)". If present, validate with `validate_musicxml()` from `engrave.musicxml.validator` and parse with `music21.converter.parse()`. Pass if both succeed.
5. `check_all_parts_present(zip_path: Path, expected_instrument_count: int)` -- Count PDF files. Expect 1 score + N parts = N+1 PDFs. Pass if count matches.
6. `check_correct_transpositions(zip_path: Path)` -- Extract .musicxml, parse with music21. For each part, check that transposing instruments (trumpet=Bb, alto sax=Eb, tenor sax=Bb, bari sax=Eb) have `<transpose>` elements with expected chromatic/diatonic values. If no `<transpose>` elements found, report as message "transposition metadata absent -- manual verification recommended" with passed=True (not a failure, per RESEARCH.md Open Question 1).
7. `check_note_count(zip_path: Path)` -- Parse MusicXML with music21. Count notes per part. Pass if every non-drum part has > 0 notes. Report per-part counts in details dict.
8. `check_pdf_file_size(zip_path: Path, min_size_bytes: int = 50_000)` -- Pass if every PDF in ZIP exceeds minimum size threshold.
9. `check_zip_file_count(zip_path: Path, expected_min: int, expected_max: int)` -- Count total files in ZIP. Pass if count is within [expected_min, expected_max] range (38-39 for big band, accounting for optional MusicXML).

For checks 4, 6, 7 that need MusicXML: if no .musicxml is in the ZIP, handle gracefully (SKIPPED for check 4, skip transposition/note checks with appropriate messages for 6 and 7).

**Unit tests:**

`tests/unit/test_smoke_checks.py`:

- Test `check_no_exceptions` with None (pass) and "error msg" (fail).
- Test `check_valid_pdfs` with ZIP containing non-empty PDFs (pass) and empty PDF (fail).
- Test `check_pdf_file_size` with PDFs above and below threshold.
- Test `check_zip_file_count` with correct and incorrect counts.
- Test `check_valid_musicxml` with no .musicxml (SKIPPED), valid .musicxml (pass -- use a minimal MusicXML string), and invalid .musicxml (fail).
- Test `check_compilable_ly` with matching .ly/.pdf pairs (pass) and missing .pdf (fail).

`tests/unit/test_smoke_runner.py`:

- Test `discover_inputs` with empty directory returns empty list.
- Test that `SmokeResult`, `InputResult`, `CheckResult` dataclasses serialize to dict/JSON correctly.

Do NOT use pytest-bdd/Gherkin for these tests -- they are straightforward unit tests.
