## Decisions Made
- Check functions return CheckResult dataclasses instead of raising exceptions, enabling uniform pass/fail/skip reporting
- MusicXML absence is SKIPPED (passed=True) not FAILED, since MusicXML generation is optional in the pipeline
- check_compilable_ly verifies that each .ly file has a corresponding .pdf in the ZIP (not re-compilation), following the anti-pattern guidance from RESEARCH.md
- Transposition check reads XML <transpose> elements directly; reports "manual verification recommended" when elements are absent (per RESEARCH.md Open Question 1)
- LLM connectivity is checked before running any inputs to fail fast with a clear message (per Pitfall 3)
- JSON output keyed by check name (dict) rather than list, matching the schema from RESEARCH.md
