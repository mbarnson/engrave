**`src/engrave/web/app.py`**: Create minimal FastAPI app following the pattern from RESEARCH.md Pattern 3:

- `JobStatus` enum: PENDING, PROCESSING, COMPLETE, FAILED.
- `jobs: dict[str, dict]` in-memory job store (single user, local machine -- per CONTEXT.md decision).
- `create_app() -> FastAPI`: Factory function that creates and returns the app. Set `docs_url=None, redoc_url=None` (no Swagger UI needed for UAT).
- `GET /` -- Serve `index.html` via Jinja2Templates. Templates directory is `Path(__file__).parent / "templates"`.
- `POST /engrave` -- Accept `file: UploadFile` and `hints: str = Form("")`. Save uploaded file to disk FIRST in the request handler using `shutil.copyfileobj(file.file, destination)` (per Pitfall 1 -- UploadFile closed after handler returns). Generate job_id as `str(uuid.uuid4())[:8]`. Create job directory `jobs/web_{job_id}/`. Store job in `jobs` dict. Start pipeline via `asyncio.create_task(_run_pipeline(...))` (NOT BackgroundTasks -- per Pitfall 2, need async event loop). Return HTML response containing the status polling div: `<div id="status" hx-get="/status/{job_id}" hx-trigger="every 3s" hx-swap="outerHTML"><p>Processing...</p></div>`. This response replaces the `#status` div via htmx.
- `GET /status/{job_id}` -- If job not found, return 404. If PROCESSING, return HTML div with `hx-get` and `hx-trigger="every 3s"` to continue polling. Include elapsed time in the message (e.g., "Processing... (2m 15s)") per Pitfall 5. If COMPLETE, return HTML with download link and HTTP status 286 (stops htmx polling). If FAILED, return HTML with error message and HTTP status 286.
- `GET /download/{job_id}` -- Return `FileResponse` serving the ZIP file with appropriate filename.
- `async _run_pipeline(job_id, input_path, hints, job_dir)`: Determine pipeline path by extension. For audio (`.wav/.mp3/.flac/.aiff`): instantiate AudioPipeline, call `process()`, then `generate_from_midi()` on stem MIDI, then `RenderPipeline.render()`. For MIDI (`.mid/.midi`): call `generate_from_midi()` then `RenderPipeline.render()`. On success, update job status to COMPLETE and set zip_path. On exception, update to FAILED with error message. Set a configurable timeout (default 30 minutes, per Pitfall 5).
- All pipeline function calls use the same imports and patterns as `cli.py` (lazy imports from `engrave.config.settings`, `engrave.llm.router`, etc.).
- Store job `start_time` (via `time.monotonic()`) so the status endpoint can report elapsed duration.

**`src/engrave/web/templates/index.html`**: Single-page HTML following RESEARCH.md Pattern 4:
- Load htmx from CDN: `https://unpkg.com/htmx.org@2.0.4`
- Clean, minimal styling with system-ui font, centered layout (max-width 600px), dashed border upload area.
- Form with `hx-post="/engrave"`, `hx-target="#status"`, `hx-swap="outerHTML"`, `hx-encoding="multipart/form-data"`.
- File input accepting `.mid,.midi,.wav,.mp3,.flac,.aiff`.
- Textarea for hints with placeholder "Optional hints (e.g., Big band, swing feel, soli at bar 17)".
- Engrave button.
- Empty `<div id="status"></div>` target for htmx responses.
- Include a brief description above the form: "Upload a MIDI file or audio recording to generate publication-quality sheet music."

**CLI `serve` command** in `cli.py`: Add after the existing benchmark commands:

```python
@app.command("serve")
def serve(
    host: str = typer.Option("127.0.0.1", help="Host to bind to"),
    port: int = typer.Option(8000, help="Port to bind to"),
) -> None:
    """Start the minimal web UI for UAT testing."""
    import uvicorn
    from engrave.web.app import create_app

    web_app = create_app()
    typer.echo(f"Starting Engrave web UI at http://{host}:{port}")
    uvicorn.run(web_app, host=host, port=port)
```
