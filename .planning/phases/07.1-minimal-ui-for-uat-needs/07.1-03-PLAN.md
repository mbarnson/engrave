  <action>
Create `TESTPLAN.md` at the project root with the following structure:

**Header:** Title ("Engrave UAT Test Plan"), date, tester name placeholder, purpose statement ("Evaluate the musical quality of Engrave output on real-world inputs").

**Prerequisites:**
- Engrave installed and `engrave --version` works
- LilyPond installed (`lilypond --version`)
- LLM provider configured (`engrave check all` passes)
- PDF viewer available (Preview on macOS)
- Optional: MuseScore or Dorico for MusicXML import testing

**Section 1: Smoke Test Pre-Validation**
- Step: Run `engrave smoke-test ./test-fixtures/` on prepared test inputs
- Expected: All structural checks pass (green checkmarks in terminal)
- If failures: Fix before proceeding to musical quality evaluation
- Note: If MusicXML is SKIPPED, that is expected (JSON fan-out is optional)

**Section 2: MIDI-Only Pipeline (Fast Path)**
For each test MIDI file:
- [ ] Run `engrave generate <file.mid> --output output/` then `engrave render output/`
- [ ] Open conductor score PDF
  - Can you read the score at conductor desk distance?
  - Are instruments in standard big band order (saxes, trumpets, trombones, rhythm)?
  - Are system brackets/braces correct?
  - Notes: ___
- [ ] Open trumpet 1 part PDF
  - Can you sight-read this part? (yes/no)
  - Are dynamics visible and correctly placed?
  - Are rehearsal marks at logical places?
  - Are multi-bar rests consolidated correctly?
  - Does it feel like standard trumpet notation? (clef, key, range)
  - Notes: ___
- [ ] Open alto sax 1 part PDF
  - Is it transposed to Eb correctly? (compare a phrase against concert pitch score)
  - Can you sight-read this part?
  - Notes: ___
- [ ] Open trombone 1 part PDF
  - Bass clef? Reading key correct (concert pitch)?
  - Notes: ___
- [ ] Open piano part PDF
  - Grand staff? Chord symbols above?
  - Notes: ___
- [ ] Import .musicxml into Dorico/MuseScore (if available)
  - Does it import without errors?
  - Do the parts match the PDFs?
  - Notes: ___

**Section 3: Audio Pipeline (Full Path)**
For each test audio file:
- [ ] Run via web UI: `engrave serve`, open browser, upload file, add hints, click Engrave
- [ ] Wait for completion (may take several minutes for audio files)
- [ ] Download ZIP, repeat Section 2 checks on the output
- [ ] Additional audio-specific checks:
  - Does the generated music resemble the source recording?
  - Are the right instruments detected?
  - Is the tempo approximately correct?
  - Notes: ___

**Section 4: Web UI Usability**
- [ ] Can you figure out how to use the web UI without instructions?
- [ ] Is the upload process clear?
- [ ] Is the "Processing..." wait tolerable? (note how long it took)
- [ ] Is the download link obvious when complete?
- [ ] If an error occurs, is the message understandable?
- [ ] Notes: ___

**Section 5: Overall Assessment**
- [ ] Would you hand these parts to your musicians for rehearsal? (yes/no/with-edits)
- [ ] What is the single biggest issue that would prevent rehearsal use?
- [ ] What surprised you positively?
- [ ] Any other notes: ___

**Footer:** "Submit this checklist with your notes to Pat. Scribble freely -- this is not a QA form."

Keep the language casual and musician-oriented. No technical jargon beyond what Sam already knows (conductor score, parts, transposition, rehearsal marks). Use checkboxes (`- [ ]`) for completion tracking and "Notes: ___" lines for free-form feedback.
